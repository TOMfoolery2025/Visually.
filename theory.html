Computer Architecture Theory

A comprehensive guide to Caches, RISC-V, and Memory Systems.

1. The Memory Hierarchy

In modern computer systems, we want memory to be fast, large, and cheap. However, no single technology satisfies all three.

The Principle of Locality: Programs tend to access a relatively small portion of their address space at any instant of time.
  • Temporal Locality: If an item is referenced, it will tend to be referenced again soon (e.g., loops).
  • Spatial Locality: If an item is referenced, items whose addresses are close by will tend to be referenced soon (e.g., arrays).

To exploit this, we build a hierarchy:

  • Registers (CPU): Instant access, very small (32 words in RISC-V).
  • L1 Cache: Very fast (1-4 cycles), small (KB), integrated into CPU core.
  • L2 Cache: Fast (10-20 cycles), medium (MB), shared or private.
  • Main Memory (DRAM): Slow (100+ cycles), large (GB).
  • Disk/SSD: Very slow (millions of cycles), huge (TB).

2. Cache Organization

How do we find data in the cache? We divide the memory address into three parts: Tag, Index, and Offset.

Direct Mapped Cache

Each memory block maps to exactly one location in the cache. Simple to build, but prone to Conflict Misses.

Index = (Block Address) % (Number of Cache Blocks)

Set Associative Cache

Each memory block can go into any of the N ways in a specific set. This reduces conflicts but increases complexity (need to search N tags).

Set Index = (Block Address) % (Number of Sets)

Fully Associative Cache

A block can be placed anywhere in the cache. No conflict misses, but requires searching every tag in the cache (expensive hardware).

3. The 3 C's of Cache Misses

  • Compulsory (Cold) Miss: The first access to a block. The block has never been in the cache before.
  • Capacity Miss: The cache cannot contain all the blocks needed during execution of a program. Blocks are discarded and later retrieved.
  • Conflict Miss (Collision): Multiple blocks compete for the same set. In Direct Mapped or Set Associative caches.

4. Write Policies

What happens when we write data?

Write-Through

Data is written to both the cache and the lower-level memory (RAM).
‚úÖ Simpler to implement. Data in RAM is always consistent.
‚ùå Slow, as writing to RAM takes time.

Write-Back

Data is written only to the cache. The block is marked as Dirty. RAM is updated only when the dirty block is replaced.
‚úÖ Faster, writes happen at cache speed.
‚ùå More complex. RAM has stale data until writeback.

5. RISC-V Context

RISC-V is a modern, open Instruction Set Architecture (ISA).

  • Load/Store Architecture: Only lb, lw, sb, sw instructions access memory. Arithmetic is done in registers.
  • Registers: 32 General Purpose Registers (x0-x31). x0 is hardwired to 0.
  • FENCE.I: An instruction used to synchronize the Instruction Cache and Data Cache (essential for self-modifying code or JIT compilers).
